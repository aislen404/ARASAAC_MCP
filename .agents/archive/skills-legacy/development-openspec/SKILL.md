# Workflow: development_openspec_codex

## Pasos

1. **intake** — Intake estructurado de necesidad social
2. **proposal** — Redactar proposal.md
3. **design** — Redactar design.md y ADRs si aplica
4. **compliance_precheck** — Precheck licencia, privacidad y seguridad
5. **atomic_tasks** — Dividir tasks.md atómicas listas para el agente de implementación
6. **codex_implementation** — Implementación según OpenSpec (Cursor Agent)
7. **tests** — Ejecutar y añadir tests
8. **review** — Review técnica, QA, accesibilidad y seguridad
9. **archive** — Archivar change OpenSpec

## Reglas transversales

- Solo pictogramas reales ARASAAC.
- Revisión humana obligatoria antes de exportar.
- Atribución visible en toda exportación.
- Sin datos personales en MVP.
- OpenSpec aprobada para cambios de producto/código.
